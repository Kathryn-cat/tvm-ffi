# TVM-FFI Traits & Text Format Design

## Goal

`@tvm_ffi.dataclasses.py_class` already lets users define MLIR-style IRs
as pure Python dataclasses — fields, structural equality, reflection, and
C++/Rust interop come for free.

The missing pieces are **semantic metadata** and **text format**.  This
document proposes a single trait decorator per class that carries both:

- **Semantic parameters** describe WHAT the IR node IS — a loop, a
  value, a function.  Consumed by all tools.
- **`text_*` parameters** describe HOW to render/parse it.  Consumed
  only by the text printer/parser.  Each `text_*` parameter is
  independently optional.

```python
@ffi.traits.For(
    # --- semantic ---
    region=ffi.traits.Region(
        def_values="$field:loop_var",
        def_expr="$field:iter_expr",
        body="$field:body",
    ),
    attrs="$field:annotations",
    # --- text format ---
    text_printer_kind="$lang.serial",
)
```

Design principles:

- **Concise.**  One decorator per class.  Don't invent abstractions when
  existing ones suffice.
- **Layered.**  Semantic fields are the foundation; `text_*` is one
  consumer.  Other consumers (optimizers, interpreters, visualizers) read
  the same semantic fields and ignore `text_*`.
- **Usable out of the box.**  A bare `@py_class` with zero annotations
  gets a working printer/parser.  `text_*` is entirely optional.

## Layer 1: Semantic Traits

Semantic trait parameters (everything except `text_*`) describe the
**role** of each IR node.  They are consumed by all tools.

### Field References

Trait parameters reference IR fields via strings with a `$` prefix:

- **`"$field:name"`** — read a field: `getattr(obj, "name")`
- **`"$method:name"`** — call a method: `getattr(obj, "name")()`
- **`"$global:name"`** — call a registered global function

```python
@ffi.traits.Func(symbol="$field:name", ...)      # plain field
@ffi.traits.Value(type="$method:printed_type")    # computed via method
```

### Trait Catalog

Every trait accepts optional `text_*` parameters for text-format
customization (see Layer 2).  Only the relevant ones are shown below.

#### `Value` — SSA Variable

```python
class Value:
    name: str                              # "$field:..." or "$method:..."
    type: str | None = None
```

#### `Region` — Scoped Body

```python
class Region:
    body: str
    def_values: str | None = None          # vars defined before body
    def_expr: str | None = None            # expression that binds def_values
    ret: str | None = None
```

`def_expr` unifies for-loops and with-statements:
- `for x in expr:` → `def_values`=loop var, `def_expr`=iterator
- `with expr as x:` → `def_values`=as-var, `def_expr`=context
- `def f(params):` → `def_values`=params, no `def_expr`

#### `Func` — Function

```python
class Func:
    symbol: str
    region: Region
    attrs: str | None = None
    text_printer_kind: str | None = None       # e.g. "$lang.prim_func"
    text_printer_pre: str | None = None        # e.g. "$method:print_prologue"
```

#### `For` — Loop

```python
class For:
    region: Region                             # must have def_values + def_expr
    attrs: str | None = None                   # loop annotations/attributes
```

#### `With` — Context Manager

```python
class With:
    region: Region                             # must have def_values + def_expr
    text_printer_kind: str | None = None       # e.g. "$lang.block"
    text_printer_pre: str | None = None
```

#### `While` — While-Loop

```python
class While:
    cond: str
    region: Region
```

#### `If` — Conditional

```python
class If:
    cond: str
    then_region: Region
    else_region: Region | None = None
```

#### `Class` — Module / Class

```python
class Class:
    symbol: str
    region: Region
    bases: str | None = None
    text_printer_kind: str | None = None       # e.g. "$lang.ir_module"
    text_printer_pre: str | None = None
```

#### `BinOp` / `UnaryOp`

```python
class BinOp:
    lhs: str
    rhs: str
    op: str                                    # "+", "-", "and", etc.

class UnaryOp:
    operand: str
    op: str                                    # "-x", "~x", "not x"
```

#### `Assign` — Assignment / Binding

```python
class Assign:
    def_values: str                            # target variable(s) being defined
    rhs: str                                   # value expression
```

Note: type annotations belong to `Value` (the variable), not to `Assign`.
The text printer may render `x: $lang.int32 = ...` by reading the variable's
`Value.type`, not from an `Assign` annotation field.

#### `StmtWithRegion` — Let-Binding

```python
class StmtWithRegion:
    def_values: str
    rhs: str
    region: Region
```

#### `Load` — Memory / Buffer Read

```python
class Load:
    source: str                                # buffer / tensor being read
    indices: str                               # index expressions
```

Printed as `source[i, j, ...]`.  Semantically: a memory read at
given indices.

Use cases:
- TIR `BufferLoad`: `A[i, j]` reading from a buffer
- MLIR `memref.load`: `memref.load %A[%i, %j]`
- Tensor indexing in any IR with array-like access

#### `Store` — Memory / Buffer Write

```python
class Store:
    target: str                                # buffer / tensor being written
    indices: str                               # index expressions
    value: str                                 # value being stored
```

Printed as `target[i, j, ...] = value`.  Semantically: a memory write.

Use cases:
- TIR `BufferStore`: `A[i, j] = value`
- TIR augmented store: `A[i] += value` (desugared to load + op + store)
- MLIR `memref.store`: `memref.store %v, %A[%i, %j]`

#### `Return`

```python
class Return:
    value: str
```

### Who Consumes Semantic Traits

| Consumer | What it reads |
|----------|--------------|
| Text printer/parser | Semantic fields + `text_*` parameters |
| Graph visualizer | `Value.name`, `Region` nesting |
| Interpreter | `For` bounds, `If` condition |
| Loop optimizer | `For.region` structure, `For.attrs` |
| Type checker | `Value.type`, `Func` return |
| Code generator | `BinOp.op`, `Load`/`Store` patterns |
| Dependence analysis | `Load.source`/`Store.target` + indices |

## Layer 2: `text_*` Parameters

Text-format-specific parameters use the `text_` prefix.  They are
ignored by all non-text consumers.

Available `text_*` parameters (all optional):

| Parameter | Purpose | Example |
|-----------|---------|---------|
| `text_printer_kind` | Decorator or callee text | `"$lang.prim_func"` |
| `text_printer_pre` | Method called before body | `"$method:print_prologue"` |
| `text_printer_post` | Method called after body | `"$method:print_outputs"` |

When no `text_*` parameters are provided, the framework infers a
default text rendering from the semantic fields alone.

## Customization Levels

### Level 0: Zero-Config Default

Every `@py_class` gets a default printer/parser with no annotations.

```python
@py_class("my_ir.Add")
class Add(Expr):
    lhs: Expr
    rhs: Expr
```

Default output: `my_ir.Add(lhs=<lhs>, rhs=<rhs>)`

### Level 1: Trait Decorators

A trait maps the IR node to a well-known syntax form.  The framework
auto-generates printer and parser:

```python
@py_class("my_ir.Add")
@ffi.traits.BinOp(lhs="$field:lhs", rhs="$field:rhs", op="+")
class Add(Expr):
    lhs: Expr
    rhs: Expr
```

Output: `<lhs> + <rhs>`

No `text_*` parameters needed — the framework knows how to print a
`BinOp`.  This is the common case for most IR nodes.

#### Custom decorator / callee via `text_printer_kind`

When the default rendering needs a dialect-specific decorator or callee:

```python
@ffi.traits.Func(
    symbol="$field:name",
    region=ffi.traits.Region(def_values="$field:params", body="$field:body"),
    text_printer_kind="$lang.prim_func",
)
```

Output: `@$lang.prim_func  def name(params...): body...`

Without `text_printer_kind`, the decorator would be the generic form.

#### Region printer methods via `text_printer_pre` / `text_printer_post`

When the region needs custom prologue/epilogue logic:

```python
@ffi.traits.Func(
    symbol="$field:name",
    region=ffi.traits.Region(def_values="$field:params", body="$field:body"),
    text_printer_kind="$lang.prim_func",
    text_printer_pre="$method:print_prologue",
)
class PrimFunc(Node):
    def print_prologue(self, printer_ctx, frame):
        """Only the prologue is custom; everything else is auto-generated."""
        if self.attrs:
            frame.stmts.append(printer_ctx.emit_call("$lang.func_attr", self.attrs))
        for var, buf in self.buffer_map.items():
            if is_simple_buffer(buf):
                ...  # inline as param annotation
            else:
                ...  # emit A = $lang.match_buffer(...)
```

Parsing doesn't need `text_printer_pre` — each `$lang.func_attr(...)` and
`$lang.match_buffer(...)` is a language module entry whose
`__ffi_text_parse__` handles parsing via value-driven dispatch.

### Level 2: Manual Override

For genuinely irregular syntax:

```python
@py_class("my_ir.CommReducer")
class CommReducer(Node):
    combiner: Function
    identity: list[Expr]

    def __ffi_text_print__(self, printer_ctx, path):
        """Instance method: full control over printing."""
        ...

    @classmethod
    def __ffi_text_parse__(cls, parser_ctx, frame: dict[str, Any]) -> "CommReducer":
        """Static factory: full control over parsing.

        Parameters
        ----------
        parser_ctx : the parser context (var_table, eval_expr, etc.)
        frame : dict of already-parsed sub-values keyed by field name

        Returns
        -------
        An instance of this class.
        """
        ...
```

- `__ffi_text_print__` is an instance method (has access to `self`).
- `__ffi_text_parse__` is a classmethod / static factory that receives
  a `parser_ctx` and a `frame` dict, and returns an instance.

Manual methods take precedence over everything.

> `__ffi_text_print__` replaces `__ir_print__`.  No backward
> compatibility — all uses should be renamed.

## Canonical Syntax

Each semantic trait maps to a well-known Python syntax pattern:

```python
# Value reference
${name}

# BinOp / UnaryOp
${lhs} + ${rhs}
-${operand}

# Load / Store
${source}[${i}, ${j}]
${target}[${i}, ${j}] = ${value}

# Assign
${def_values} = ${rhs}

# Return
return ${value}

# Func
@${text_printer_kind}(${attrs...})
def ${symbol}(${def_values...}):
    ${prologue...}
    ${body}

# For
for ${def_values} in ${def_expr}:
    ${body}

# With
with ${def_expr} as ${def_values}:
    ${prologue...}
    ${body}

# If
if ${cond}:
    ${then_body}
else:
    ${else_body}

# While
while ${cond}:
    ${body}

# Class
@${text_printer_kind}(${attrs...})
class ${symbol}(${bases...}):
    ${prologue...}
    ${members}
```

## Parsing: Value-Driven Dispatch

The parser evaluates expressions first, then dispatches on the result:

```
Parser sees:    with $lang.block("name") as [vi, vj]: ...
Step 1: Eval    $lang.block("name")  →  BlockFrame
Step 2: Call    BlockFrame.__ffi_text_parse__(parser_ctx, with_node)
```

Each language-module entry returns a **surface object** whose
`__ffi_text_parse__` handles the surrounding AST context.  This replaces
TVM's global dispatch table and isinstance chains.

Note: the surface object's `__ffi_text_parse__(parser_ctx, ast_node)` is
distinct from the IR class's Level 2 override
`__ffi_text_parse__(cls, parser_ctx, frame)`.  The surface object parses
the AST and constructs the IR node; the IR class override customizes how
a parsed frame is assembled into the final object.

### Parser Core

```python
class IRParser:
    def __init__(self):
        self.var_table = VarTable()
        self.syntax_ctx = SyntaxContext()

    # Value-driven visitors
    def visit_For(self, node):
        return self._dispatch(self.eval_expr(node.iter), node)
    def visit_With(self, node):
        if len(node.items) == 1:
            return self._dispatch(
                self.eval_expr(node.items[0].context_expr), node)
        return self._multi_with(node)
    def visit_Assign(self, node):
        if _is_subscript_lhs(node):
            return self._subscript_store(node)
        return self._dispatch_or_bind(self.eval_expr(node.value), node)
    def visit_AnnAssign(self, node):
        return self._dispatch_or_bind(self.eval_expr(node.value), node)
    def visit_FunctionDef(self, node):
        for dec in node.decorator_list:
            if r := self._try_dispatch(self.eval_expr(dec), node):
                return r
        raise ParseError("Unrecognized decorator")
    def visit_ClassDef(self, node):
        for dec in node.decorator_list:
            if r := self._try_dispatch(self.eval_expr(dec), node):
                return r
        raise ParseError("Unrecognized decorator")
    def visit_Expr(self, node):
        self._try_dispatch(self.eval_expr(node.value), node)

    # Structural visitors (use SyntaxContext)
    def visit_If(self, node):     return self.syntax_ctx.handle_if(self, node)
    def visit_While(self, node):  return self.syntax_ctx.handle_while(self, node)
    def visit_Return(self, node): return self.syntax_ctx.handle_return(self, node)
    def visit_AugAssign(self, node): ...  # desugar to load+op+store
    def visit_Continue(self, node):  ...
    def visit_Break(self, node):     ...
    def visit_Pass(self, node):      pass

    # Dispatch
    def _dispatch(self, val, node):
        if hasattr(val, "__ffi_text_parse__"):
            return val.__ffi_text_parse__(self, node)
        raise ParseError(f"Expected IR construct, got {type(val)}")
    def _dispatch_or_bind(self, val, node):
        if hasattr(val, "__ffi_text_parse__"):
            return val.__ffi_text_parse__(self, node)
        self._default_bind(val, node)
    def _try_dispatch(self, val, node):
        if hasattr(val, "__ffi_text_parse__"):
            return val.__ffi_text_parse__(self, node)
        return None
```

### `SyntaxContext`

Structural constructs (`if`, `while`, `return`) differ by dialect.
`SyntaxContext` is pushed when entering a region:

```python
@dataclass
class SyntaxContext:
    handle_if: Callable
    handle_while: Callable
    handle_return: Callable
    handle_assert: Callable
    scope_callees: set[str]
```

Derived from the enclosing trait's `text_*` parameters.

---

## Printing

1. Look up the node's trait.
2. Print header (using `text_printer_kind` if set, else default).
3. If the trait has a `Region`:
   a. Define `def_values`.
   b. Call `text_printer_pre` method (if set).
   c. Print `body`.
   d. Handle `ret`.
   e. Call `text_printer_post` method (if set).
4. Recurse into children.

The printer is a **canonicalizer**.

```python
            frame.stmts.append(printer_ctx.emit_call("T.func_attr", self.attrs))
        for var, buf in self.buffer_map.items():
            if is_simple_buffer(buf):
                ...
            else:
                frame.stmts.append(...)  # A = T.match_buffer(...)
```

### TIR SBlock

```python
@py_class("tir.SBlock")
@ffi.traits.With(
    region=ffi.traits.Region(def_values="$field:iter_vars", body="$field:body"),
    text_printer_kind="T.block",
    text_printer_pre="$method:print_block_header",
)
class SBlock(Node):
    def print_block_header(self, printer_ctx, frame):
        frame.stmts.append(printer_ctx.emit_call("T.reads", self.reads))
        frame.stmts.append(printer_ctx.emit_call("T.writes", self.writes))
        for buf in self.alloc_buffers:
            frame.stmts.append(printer_ctx.emit_assign(buf.name, "T.alloc_buffer", ...))
        if self.init is not None:
            frame.stmts.append(printer_ctx.emit_with("T.init", self.init))
```

## Sugar vs Canonical

Traits generate **canonical syntax** — the stable, roundtrip-safe form.

| Sugar | Why not canonical |
|-------|-------------------|
| `$lang.macro` | macro body inlined; definition lost |
| `$lang.meta_var` | parse-time-only; not in final IR |
| `$lang.emit_te` | desugared to `call_tir` |
| `$lang.comm_reducer(lambda ...)` | lambda lowered away |
| `$lang.grid(4, 4)` | desugared to nested serial loops |

Supported via Level 2 manual overrides, not core traits.

## Language Modules

Each dialect provides a language module — the namespace in printed text:

```python
@ffi.lang("T")
class T:
    prim_func    = ffi.lang.decorator(PrimFunc)
    serial       = ffi.lang.for_iter(TIRFor, kind="serial")
    parallel     = ffi.lang.for_iter(TIRFor, kind="parallel")
    block        = ffi.lang.with_ctx(SBlock)
    match_buffer = ffi.lang.binding(MatchBufferRegion)
    Buffer       = ffi.lang.proxy(BufferProxy)
    int32        = ffi.lang.dtype("int32")
```

Each entry returns a surface object with `__ffi_text_parse__`,
auto-generated from the trait.  `.pyi` stubs via `tvm-ffi-stubgen`.

## Summary

| Layer | Concern | Mechanism |
|-------|---------|-----------|
| Semantic | What the IR node IS | `@ffi.traits.For(region=...)` |
| Semantic | Field references | `"$field:name"` / `"$method:name"` / `"$global:name"` |
| Text L0 | Default printer/parser | Auto from `@py_class` fields |
| Text L1 | Syntax mapping | Trait decorator (default rendering) |
| Text L1 | Custom decorator/callee | `text_printer_kind="$lang.prim_func"` |
| Text L1 | Region prologue/epilogue | `text_printer_pre="$method:..."` |
| Text L2 | Irregular syntax | `__ffi_text_print__` / `__ffi_text_parse__` |
| Text | Structural rules | `SyntaxContext` (from `text_*` params) |
| Text | Parser dispatch | `result.__ffi_text_parse__(parser_ctx, node)` |
| Text | Language module | `@ffi.lang` + `tvm-ffi-stubgen` |
