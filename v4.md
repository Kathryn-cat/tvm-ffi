# TVM-FFI Traits & Text Format Design

## Goal

`@tvm_ffi.dataclasses.py_class` already gives us the structural IR
definition: fields, reflection, equality, and C++/Rust interop.

This design adds:

1. **Semantic traits** — a small vocabulary describing WHAT each IR node IS
   (a loop, a value, a function).  Consumed by all tools.
2. **Auto-generated printer/parser** — for most `@py_class` IR nodes, a
   one-line trait decorator produces both `__ffi_text_print__` and
   `__ffi_text_parse__`.
3. **Clean escape hatch** — genuinely irregular syntax gets a manual
   override without polluting the core.

### Scope

The text format is **Python-style** — printed IR is valid Python that can
be edited with IDE support.  This is the same format family as TVM's
TVMScript.

**Not in scope**: MLIR assembly syntax (`%0 = arith.addi %1, %2 : i32`).
MLIR operations that want MLIR-native text should use MLIR's ODS and
`assemblyFormat`.  The two systems complement each other — a project can
use both.

### Design principles

- **Concise.**  One decorator per class.  No configuration DSLs.
- **Layered.**  Semantic traits are the foundation; `text_*` is one
  consumer.  Other consumers (optimizers, visualizers) read traits and
  ignore `text_*`.
- **Honest.**  The auto-generated printer targets canonical syntax
  recoverable from the final IR.  Sugar is a separate layer.
- **Usable out of the box.**  `@py_class` with zero annotations gets a
  working printer/parser.  Traits and `text_*` are optional.

### Validated against

- TVM TIRx: `3rdparty/tvm/python/tvm/tirx/`, printer in
  `3rdparty/tvm/src/script/printer/tirx/` (7 files, ~2500 LOC), parser
  in `3rdparty/tvm/python/tvm/script/parser/tirx/` (15 dispatch handlers)
- TVM Relax: `3rdparty/tvm/python/tvm/relax/`, printer in
  `3rdparty/tvm/src/script/printer/relax/` (7 files, ~2000 LOC), parser
  in `3rdparty/tvm/python/tvm/script/parser/relax/` (14 dispatch handlers)
- MLIR: `assemblyFormat` + ODS in `/llvm-project/mlir/`, custom
  printer/parser fallback patterns in SCF, Linalg, Func dialects
- cuteDSL: `/dynamic-kernel-generator/DkgDSL/`, Python DSL with operator
  overloading + AST preprocessing targeting MLIR

## Trait Catalog

### Field References

Trait parameters reference IR fields via `$`-prefixed strings:

- **`"$field:name"`** — read a field: `getattr(obj, "name")`
- **`"$method:name"`** — call a method: `getattr(obj, "name")()`
- **`"$global:name"`** — call a registered global function

`$method:` is the standard approach when the printed value is computed
from multiple fields (common for loop bounds, type annotations, etc.).

```python
@ffi.traits.For(
    region=ffi.traits.Region(def_values="$field:loop_var", body="$field:body"),
    start="$field:min",
    end="$method:computed_end",            # = min + extent
    step="$field:step",
    text_printer_kind="$method:kind_prefix",   # T.serial / T.parallel / ...
)
class TIRFor(Stmt):
    @property
    def computed_end(self):
        return self.min + self.extent

    @property
    def kind_prefix(self):
        return {ForKind.SERIAL: "T.serial", ...}[self.kind]
```

### `Region` — Scoped Body

```python
class Region:
    body: str                              # "$field:..." or "$method:..."
    def_values: str | None = None          # vars defined before body
    def_expr: str | None = None            # expression that binds def_values
    ret: str | None = None                 # return / output value
```

`def_expr` is the expression after `in` or `as` in Python:
- `with expr as x:` → `def_values`=as-var, `def_expr`=context expression
- `def f(params):` → `def_values`=params, no `def_expr`

For **for-loops**, use the `For` trait's `start`/`end`/`step` fields
instead of `def_expr` (see below).

### `Value` — SSA Variable

```python
class Value:
    name: str                              # "$field:..." or "$method:..."
    type_ann: str | None = None            # type annotation at def site
```

### `Func` — Function

```python
class Func:
    symbol: str
    region: Region
    attrs: str | None = None
    text_printer_kind: str | None = None   # e.g. "$lang.prim_func"
    text_printer_pre: str | None = None    # e.g. "$method:print_prologue"
```

### `For` — Loop

```python
class For:
    region: Region                         # must have def_values + body
    # Range bounds (the common case for structured loops):
    start: str | None = None               # "$field:..." — default: 0
    end: str | None = None                 # "$field:..." or "$method:..."
    step: str | None = None                # "$field:..." — default: 1
    # Loop-carry reduction (values carried across iterations):
    def_carry: str | None = None           # carried variables defined at loop entry
    carry_init: str | None = None          # initial values for first iteration
    attrs: str | None = None
    text_printer_kind: str | None = None   # overrides default "range"
```

When `end` is provided, the printer auto-constructs the iterator
expression using Python's `range(start, end, step)` convention, with
default elision:

| Fields set | Printed as |
|------------|-----------|
| `end` only | `range(end)` |
| `start` + `end` | `range(start, end)` |
| `start` + `end` + `step` | `range(start, end, step)` |
| any + `text_printer_kind` | `kind(start, end, step=step, ...)` |

When `end` is NOT provided, the printer falls back to `region.def_expr`
for generic iteration (e.g. `for x in collection:`).

The `(start, end, step)` triple is **semantic** — consumed by all tools:
- Loop optimizers read bounds directly from the trait
- Dependence analysis uses bounds for iteration space reasoning
- The text printer uses bounds to construct the range expression

**Loop-carry** (`def_carry` / `carry_init`): When present, the loop
carries values across iterations.  `carry_init` provides the initial
values from the parent scope; `def_carry` names the variables defined
inside the body that receive the carried values.  The body must end
with `region.ret` (printed as `yield`) to provide the next iteration's
values.  After the loop, the final carried values become the loop's
results.

The printer emits an explicit definition line at the top of the body
for each carry variable, using `${lang}.def_carry(init_value)`:

```python
# Without carry (simple loop):
for i in range(n):
    body

# With carry (reduction loop):
for i in T.serial(range(n)):
    acc: T.float32 = T.def_carry(init)   # carry var def-site
    new_acc = acc + arr[i]
    yield new_acc
# acc holds the final reduction result
```

The `${lang}.def_carry(init)` call is the **def-site** — it makes the carry
variable visible and typed inside the body.  The parser dispatches on
`${lang}.def_carry` to create the carry binding.

### `With` — Context Manager

```python
class With:
    region: Region                         # must have def_values + def_expr
    # Scope-carry (values carried through a scoped region):
    def_carry: str | None = None           # carried variables defined at scope entry
    carry_init: str | None = None          # initial values from parent scope
    text_printer_kind: str | None = None
    text_printer_pre: str | None = None
    text_printer_post: str | None = None
```

When `def_carry` / `carry_init` are present, the scoped region carries
values through.  Same semantics as `For` carry: the printer emits
`${lang}.def_carry(init)` lines at the top of the body, and `region.ret`
provides the output via `yield`.

### `BinOp` / `UnaryOp`

```python
class BinOp:
    lhs: str
    rhs: str
    op: str                                # "+", "-", "and", etc.

class UnaryOp:
    operand: str
    op: str                                # "-", "~", "not"
```

### `Assign` — Assignment / Binding

```python
class Assign:
    def_values: str                        # target variable(s)
    rhs: str                               # value expression
```

Type annotations are from `Value.type_ann` on the target, not from Assign.

### `Load` / `Store` — Memory Access

```python
class Load:
    source: str                            # buffer/tensor
    indices: str | None = None             # index expressions (None for scalar/spatial)

class Store:
    target: str
    value: str
    indices: str | None = None             # index expressions (None for scalar/spatial)
```

When `indices` is provided, printed as `source[i, j]` / `target[i, j] = value`.
When `indices` is None, printed as `source` / `target = value` (scalar access).

### `ElementWise` — Body-Region Computation

```python
class ElementWise:
    inputs: str                            # "$field:..." — input operands
    outputs: str | None = None             # "$field:..." — output operands
    region: Region                         # body: per-element computation
    attrs: str | None = None
    text_printer_kind: str | None = None
```

For operations where a body region describes per-element or per-tile
computation over input/output operands.  Covers the linalg-style
`ins/outs { body }` pattern common in MLIR-based stacks.

Printed as:
```python
with lir.generic(inputs, outputs) as (in_elem, out_elem):
    out_elem[...] = f(in_elem[...])
```

### `Assert` — Assertion

```python
class Assert:
    cond: str                              # "$field:..." — condition expression
    message: str | None = None             # "$field:..." or "$method:..." — structured message
```

Printed as `assert <cond>, <message>`.  The parser delegates to
`SyntaxContext.handle_assert`, which determines dialect-specific
semantics (TIR's structured `(kind, [parts])` tuple vs Relax's simple
string message).

### `Return`

```python
class Return:
    value: str
```

### Who Consumes Semantic Traits

| Consumer | What it reads |
|----------|--------------|
| Text printer/parser | Semantic fields + `text_*` parameters |
| Graph visualizer | `Value.name`, `Region` nesting |
| Interpreter | `For.start`/`end`/`step`, `If` condition, `Assert` |
| Loop optimizer | `For.start`/`end`/`step`, `For.def_carry`, `For.attrs` |
| Reduction analysis | `For.def_carry`/`carry_init`, `ElementWise` patterns |
| Type checker | `Value.type_ann`, `Func` return |
| Code generator | `BinOp.op`, `Load`/`Store`, `ElementWise` patterns |
| Dependence analysis | `Load.source`/`Store.target` + indices |

## `text_*` Parameters

Text-format-specific parameters use the `text_` prefix and are ignored
by all non-text consumers.

| Parameter | Purpose | Example |
|-----------|---------|---------|
| `text_printer_kind` | Decorator/callee text | `"$lang.prim_func"` |
| `text_printer_pre` | Method called before body | `"$method:print_prologue"` |
| `text_printer_post` | Method called after body | `"$method:print_outputs"` |

When no `text_*` parameters are provided, the framework infers a default
rendering from the semantic trait alone.

## Customization Levels

The printer/parser suptive — a higher level never invalidates a lower one.

### Level 0: Zero-Config Default

Every `@py_class` (and `@c_class`) gets a working printer/parser for
free.  The printer renders `TypeKey(field1=value1, field2=value2, ...)`
using the reflection field list.  No annotations needed.

```python
@py_class("my_ir.Add")
class Add(Expr):
    lhs: Expr
    rhs: Expr
```

Output: `my_ir.Add(lhs=<lhs>, rhs=<rhs>)`

This is a **debug/dump** format — functional, roundtrip-safe, but not
human-friendly for large programs.

**When to stay at Level 0**: Leaf nodes, literals, type descriptors,
call-like expressions — anything where `name(field=value)` is already
readable.

TIRx Level 0 nodes: `Evaluate`, `SeqStmt`, `Cast`, `Select`, `IntImm`,
`FloatImm`, `Ramp`, `Broadcast`, `Let`, `Reduce` (10 nodes, 0 custom
lines).

Relax Level 0 nodes: `Tuple`, `TupleGetItem`, `ShapeExpr`, `Constant`,
`PrimValue`, `StringImm`, `DataTypeImm`, `BindingBlock`, `ExternFunc`
(9 nodes, 0 custom lines).

### Level 1: Trait Decorators

A trait maps the IR node to a well-known Python syntax pattern.  The
framework auto-generates the printer and parser.  There are two
sub-levels:

#### Level 1a: Pure trait (zero custom code)

The trait decorator is all that's needed:

```python
@py_class("tir.Add")
@ffi.traits.BinOp(lhs="$field:a", rhs="$field:b", op="+")
class Add(PrimExpr):
    a: PrimExpr
    b: PrimExpr
```

Output: `<a> + <b>`.  No methods, no hooks, no `text_*` parameters.

Other examples that are pure one-liners:

```python
@ffi.traits.Value(name="$field:name_hint", type_ann="$method:printed_type")
class Var(PrimExpr): ...

@ffi.traits.Load(source="$field:buffer", indices="$field:indices")
class BufferLoad(PrimExpr): ...

@ffi.traits.Store(target="$field:buffer", indices="$field:indices", value="$field:value")
class BufferStore(Stmt): ...

@ffi.traits.If(cond="$field:condition",
    then_region=ffi.traits.Region(body="$field:then_case"),
    else_region=ffi.traits.Region(body="$field:else_case"))
class IfThenElse(Stmt): ...

@ffi.traits.While(cond="$field:condition",
    region=ffi.traits.Region(body="$field:body"))
class While(Stmt): ...

@ffi.traits.Assign(def_values="$field:var", rhs="$field:value")
class Bind(Stmt): ...

@ffi.traits.Assert(cond="$field:condition", message="$method:structured_msg")
class AssertStmt(Stmt):
    def structured_msg(self):
        return (self.error_kind, self.message_parts)
```

TIRx Level 1a nodes: 14 BinOps, `Not`, `Var`/`SizeVar`,
`BufferLoad`, `BufferStore`, `Bind`, `While`, `IfThenElse`, `AssertStmt`
(21 nodes, 0 custom lines).

Relax Level 1a nodes: `Var`/`DataflowVar`, `Function`, `If`,
`VarBinding` (4 nodes, 0 custom lines).

#### Level 1b: Trait + `$method:` or `text_*` hooks

The trait handles the skeleton (scope, variables, body).  A small
method provides dialect-specific logic for parts the trait can't infer:

```python
@py_class("tir.For")
@ffi.traits.For(
    region=ffi.traits.Region(def_values="$field:loop_var", body="$field:body"),
    start="$field:min",
    end="$method:computed_end",
    step="$field:step",
    attrs="$field:annotations",
    text_printer_kind="$method:kind_prefix",
)
class TIRFor(Stmt):
    @property
    def computed_end(self):
        return self.min + self.extent

    @property
    def kind_prefix(self):
        return {ForKind.SERIAL: "T.serial", ForKind.PARALLEL: "T.parallel",
                ForKind.VECTORIZED: "T.vectorized", ForKind.UNROLLED: "T.unroll",
                ForKind.THREAD_BINDING: "T.thread_binding"}[self.kind]
```

Custom code: **~10 lines** (two properties).  Framework handles loop
variable binding, scope, body printing.

```python
@py_class("tir.PrimFunc")
@ffi.traits.Func(
    symbol="$field:name",
    region=ffi.traits.Region(def_values="$field:params", body="$field:body"),
    text_printer_kind="T.prim_func",
    text_printer_pre="$method:print_prologue",
)
class PrimFunc(Node):
    def print_prologue(self, ctx, frame):
        if self.attrs:
            frame.add(ctx.call_stmt("T.func_attr", self.attrs))
        for var, buf in self.buffer_map.items():
            if is_simple_buffer(buf):
                pass  # inlined in param annotation
            else:
                frame.add(ctx.assign_stmt(buf.name, "T.match_buffer", var, buf.shape, buf.dtype))
```

Custom code: **~25 lines** (prologue method + `is_simple_buffer`).
Framework handles `@T.prim_func` decorator, parameter list, body, scope.

```python
@py_class("tir.SBlock")
@ffi.traits.With(
    region=ffi.traits.Region(def_values="$field:iter_vars", body="$field:body"),
    text_printer_kind="T.block",
    text_printer_pre="$method:print_block_header",
)
class SBlock(Node):
    def print_block_header(self, ctx, frame):
        frame.add(ctx.call_stmt("T.reads", self.reads))
        frame.add(ctx.call_stmt("T.writes", self.writes))
        for k, v in self.annotations.items():
            frame.add(ctx.call_stmt("T.sblock_attr", {k: v}))
        for buf in self.match_buffers:
            frame.add(ctx.assign_stmt(buf.name, "T.match_buffer", buf.source))
        for buf in self.alloc_buffers:
            frame.add(ctx.assign_stmt(buf.name, "T.sblock_alloc_buffer", buf.shape, buf.dtype))
        if self.init is not None:
            frame.add(ctx.with_stmt("T.init", self.init))
```

Custom code: **~40 lines** (the 8 ordered clauses).  Framework handles
`with T.block(...) as [vi, vj]:`, scope, body printing.

```python
@py_class("relax.DataflowBlock")
@ffi.traits.With(
    region=ffi.traits.Region(body="$field:bindings"),
    text_printer_kind="R.dataflow",
    text_printer_post="$method:print_outputs",
)
class DataflowBlock(Node):
    def print_outputs(self, ctx, frame):
        outputs = [b.var for b in self.bindings if not isinstance(b.var, DataflowVar)]
        if outputs:
            frame.add(ctx.call_stmt("R.output", *outputs))
```

Custom code: **~8 lines**.  Framework handles `with R.dataflow():` and body.

```python
@py_class("relax.MatchCast")
@ffi.traits.Assign(def_values="$field:var", rhs="$method:printed_rhs")
class MatchCast(Node):
    def printed_rhs(self):
        return call("R.match_cast", self.value, self.struct_info)
```

Custom code: **~3 lines**.  Framework handles variable definition and
assignment.

TIRx Level 1b: `PrimFunc`, `For`, `SBlock`, `IterVar` (4 nodes, ~80
lines).

Relax Level 1b: `DataflowBlock`, `MatchCast` (2 nodes, ~11 lines).

### Level 2: Manual Override

For genuinely irregular syntax where no trait's skeleton fits.
`__ffi_text_print__` (instance method) and `__ffi_text_parse__`
(classmethod) take full control.  The framework still provides the
`ctx` (IRPrinter context) with convenience methods that keep L2 code
concise.

#### `ctx` Convenience Methods

Five methods on the printer context minimize boilerplate in L2 code:

**`ctx.call(callee, *args, **kwargs)`** — builds a Call AST node.
Each arg and kwarg value is auto-printed through the printer.  Kwargs
whose value is `ctx.ELIDED` are silently dropped.

**`ctx.elide(value, default)`** — returns `ctx.ELIDED` if `value`
equals `default`, otherwise returns `value`.  Used with `ctx.call` to
skip kwargs that match their defaults (e.g. `dtype="float32"`,
`scope="global"`).

**`ctx.lambda_(params, body)`** — builds a Lambda AST node.  Defines
`params` as variables in a fresh scope, prints `body` expression(s).
Returns the complete `LambdaDoc`.

**`ctx.scope(callee, *args, body, as_var=None, **kwargs)`** — builds a
with-statement scope: `with callee(*args, **kwargs) as as_var: body`.

**`ctx.assign(target, rhs, type_ann=None)`** — defines `target` as a
new variable and builds `target = rhs` (or `target: type_ann = rhs`).

#### TIRx Level 2 Examples

```python
@py_class("tir.CommReducer")
class CommReducer(Node):
    def __ffi_text_print__(self, ctx, path):
        return ctx.call("T.comm_reducer",
            ctx.lambda_(self.lhs + self.rhs, self.result),
            self.identity_element)
```

```python
@py_class("tir.AllocBuffer")
class AllocBuffer(Stmt):
    def __ffi_text_print__(self, ctx, path):
        ctx.define_alias(self.buffer.data, ctx.print_expr(self.buffer).attr("data"))
        return ctx.assign(self.buffer, ctx.call("T.alloc_buffer",
            self.buffer.shape,
            dtype=ctx.elide(self.buffer.dtype, "float32"),
            scope=ctx.elide(self.buffer.scope, "global"),
            annotations=ctx.elide(self.annotations, {})))
```

```python
@py_class("tir.AttrStmt")
class AttrStmt(Stmt):
    def __ffi_text_print__(self, ctx, path):
        if self.attr_key in ("thread_extent", "virtual_thread") \
                and isinstance(self.node, IterVar):
            return ctx.scope("T.launch_thread",
                self.node.thread_tag, self.value,
                body=self.body, as_var=self.node.var)
        return ctx.scope("T.attr",
            self.node, self.attr_key, self.value, body=self.body)
```

| Node | Lines | Why manual |
|------|-------|-----------|
| `AllocBuffer` | ~5 | `ctx.call` + `ctx.elide` for buffer params |
| `DeclBuffer` | ~4 | Same pattern as AllocBuffer |
| `AttrStmt` | ~10 | Basic `ctx.scope` + launch_thread variant |
| `CommReducer` | ~3 | `ctx.lambda_` for combiner reconstruction |

#### Relax Level 2 Examples

```python
@py_class("relax.Call")
class RelaxCall(Expr):
    def __ffi_text_print__(self, ctx, path):
        for handler in [self._call_tir, self._assert_op,
                        self._hint_on_device, self._print_op,
                        self._to_vdevice]:
            if (r := handler(ctx, path)) is not None:
                return r
        returcall(ctx, path)

    def _call_tir(self, ctx, path):
        if not self.op.same_as(CALL_TIR_OP):
            return None
        return ctx.call("R.call_tir",
            self.args[0], self.args[1],
            out_sinfo=self.sinfo_args[0],
            tir_vars=ctx.elide(
                self.args[2] if len(self.args) > 2 else None, None))

    def _generic_call(self, ctx, path):
        callee = self._resolve_callee(ctx)
        return ctx.call(callee, *self.args,
            **ctx.attrs_kwargs(self.attrs),
            sinfo_args=ctx.elide(
                tuple(self.sinfo_args) if self.sinfo_args else (), ()))
```

```python
@py_class("relax.TensorStructInfo")
class TensorStructInfo(StructInfo):
    def __ffi_text_print__(self, ctx, path):
        args = [self._shape_tuple(ctx)] if self.shape else []
        return ctx.call("R.Tensor", *args,
            dtype=ctx.elide(self.dtype, DataType("void")),
            ndim=ctx.elide(self.ndim if not self.shape else -1, -1),
            vdevice=ctx.elide(self.vdevice, None))
```

**Relax Level 2 nodes** (2 groups, ~70 lines):

| Node | Lines | Why manual |
|------|-------|-----------|
| `Call` | ~40 | 6 call variants with different kwarg structures |
| StructInfo (6 types) | ~30 | Each type has conditional kwarg elision |

#### Framework Conventions (0 custom lines)

Two patterns are handled entirely by the framework, requiring no
per-node code:

**Transparent wrappers**: When a node's only purpose is to carry extra
context for an inner node (e.g. `SBlockRealize` wraps `SBlock` with
`iter_values` and `predicate`), the printer delegates to the inner
node and passes the wrapper's extra fields as context.  No
`__ffi_text_print__` needed on the wrapper.

**Compound body flattening**: When a `Func` trait's `region.body`
resolves to a compound expression (e.g. Relax's `SeqExpr` containing
binding blocks + final expression), the printer flattens the blocks
inline and emits the final expression as a `return` statement.  This
eliminates `SeqExpr` from the per-node override list.

### Effort Summary

| | Level 0 | Level 1a | Level 1b | Level 2 | Total custom |
|-|---------|----------|----------|---------|-------------|
| **TIRx** (32 nodes) | 10 nodes, 0 lines | 21 nodes, 0 lines | 4 nodes, ~80 lines | 5 nodes, ~34 lines | **~114 lines** |
| **Relax** (18 nodes) | 9 nodes, 0 lines | 4 nodes, 0 lines | 2 nodes, ~11 lines | 2 nodes, ~70 lines | **~81 lines** |
| **Combined** | **19** | **25** | **6** | **7** | **~195 lines** |

vs current C++ printer: **3,669 lines** → **~95% reduction**.

The savings come from:
- **Level 1a** (25 nodes): one-line trait decorator replaces ~20 lines
  of C++ dispatch + AST construction per node
- **Level 1b** (6 nodes): framework handles scope/vars/body/decorator;
  custom code is only the dialect-specific prologue/epilogue
- **Level 2** (7 nodes): `ctx.call`/`ctx.elide`/`ctx.lambda_`/`ctx.scope`
  reduce each override to 3–12 lines
- **Framework conventions** (SBlockRealize, SeqExpr): handled by the
  printer without any per-node code
- **Infrastructure** (~900 lines of C++ utils, frames, dispatch tables):

## Printer Implementation

### Traits as `TypeAttrColumn`

Each `ffi.traits.*` decorator is stored as a **`TypeAttrColumn`** in
the TVM FFI type system.  `TypeAttrColumn` is a per-type-index lookup
table that works across both `@py_class` and `@c_class`:

```
TypeAttrColumn("__ffi_traits__")    →  trait data per type index
TypeAttrColumn("__ffi_text_print__") →  manual print override per type index
s means:
- A C++ class registered via `@c_class` can have traits set from C++
  reflection (`ObjectDef<T>().def_attr("__ffi_traits__", ...)`)
- A Python class registered via `@py_class` can have traits set from
  the `@ffi.traits.*` decorator
- The printer dispatches uniformly on the type index, regardless of
  language origin

### Tiered Dispatch

The printer resolves each node through a three-tier lookup:

```
Tier 1:  TypeAttrColumn("__ffi_text_print__")  →  manual override (Level 2)
Tier 2:  TypeAttrColumn("__ffi_traits__")      →  trait-driven print (Level 1)
Tier 3:  (fallback)                            →  default from @py_class fields (Level 0)
```

```python
def print_node(node, ctx, path):
    type_index = node.__tvm_ffi_type_index__

    # Tier 1: Manual override (__ffi_text_print__)
    text_print_col = TypeAttrColumn("__ffi_text_print__")
    manual_fn = text_print_col[type_index]
    if manual_fn is not None:
        return manual_fn(node, ctx, path)

    # Tier 2: Trait-driven
    traits_col = TypeAttrColumn("__ffi_traits__")
    trait = traits_col[type_index]
    if trait is not None:
        return trait_print(node, trait, ctx, path)

    # Tier 3: Default (Level 0)
    return default_print(node, ctx, path)
```

Level 0 `default_print` renders `TypeKey(field1=value1, field2=value2)`
using the reflection field list from `@py_class` or `@c_class`.

The printer is a **canonicalizer** — it prints the stable syntax defined
by the trait, not historical spelling.

### Per-Trait Printing

**BinOp / UnaryOp** — inline expression:

```python
def print_binop(node, trait, ctx, path):
    lhs = ctx.print_expr(resolve(trait.lhs, node), path.attr("lhs"))
    rhs = ctx.print_expr(resolve(trait.rhs, node), path.attr("rhs"))
    return ast.BinOp(lhs, trait.op, rhs)
```

**Value** — name at use site, `name: type_ann` at def site:

```python
def print_value_def(node, trait, ctx, path):
    name = resolve(trait.name, node)
    var_id = ctx.define_var(name, node)
    if trait.type_ann:
        type_expr = ctx.print_expr(resolve(trait.type_ann, node), path.attr("type_ann"))
        return ast.AnnAssign(var_id, type_expr)
    return var_id
```

**Load / Store** — subscript or scalar access:

```python
def print_load(node, trait, ctx, path):
    source = ctx.print_expr(resolve(trait.source, node), path.attr("source"))
    if trait.indices is not None:
        indices = ctx.print_exprs(resolve(trait.indices, node), path.attr("indices"))
        return ast.Index(source, indices)
    return source  # scalar access
```

**Func** — function definition with optional decorator:

```python
def print_func(node, trait, ctx, path):
    # 1. Decorator
    decorator = resolve_kind(trait.text_printer_kind, ctx) if trait.text_printer_kind else None
    # 2. Symbol and params
    symbol = resolve(trait.symbol, node)
    params = resolve(trait.region.def_values, node)
    # 3. Enter scope
    with ctx.new_frame() as frame:
        param_docs = [print_value_def(p, get_trait(p), ctx, ...) for p in params]
        # 4. Prologue
        if trait.text_printer_pre:
            resolve_method(trait.text_printer_pre, node)(ctx, frame)
        # 5. Body
        body = resolve(trait.region.body, node)
        for stmt in (body if isinstance(body, list) else [body]):
            frame.add(print_node(stmt, ctx, ...))
        # 6. Return
        if trait.region.ret:
            ret_val = resolve(trait.region.ret, node)
            frame.add(ast.Return(ctx.print_expr(ret_val, ...)))
    return ast.Function(symbol, param_docs, frame.stmts, der)
```

**For** — for-loop with range bounds:

```python
def print_for(node, trait, ctx, path):
    with ctx.new_frame() as frame:
        loop_var = resolve(trait.region.def_values, node)
        lhs = print_value_def(loop_var, get_trait(loop_var), ctx, ...)
        if trait.end is not None:
            rhs = build_range_call(
                start=resolve_or_default(trait.start, node, default=0),
                end=resolve(trait.end, node),
                step=resolve_or_default(trait.step, node, default=1),
                kind=resolve_or_default(trait.text_printer_kind, node, default=None),
                attrs=resolve_or_default(trait.attrs, node, default=None),
                ctx=ctx,
            )
        else:
            rhs = ctx.print_expr(resolve(trait.region.def_expr, node), ...)
        body = resolve(trait.region.body, node)
        for stmt in (body if isinstance(body, list) else [body]):
            frame.add(print_node(stmt, ctx, ...))
    return ast.For(lhs, rhs, frame.stmts)

def build_range_call(start, end, step, kind, attrs, ctx):
    """Construct range() or kind(start, end, ...) with default elision."""
    if kind is not None:
        callee = ctx.print_expr(kind)
        args = [ctx.print_expr(start), ctx.print_expr(end)]
        kwargs = {}
        if step != 1:
            kwargs["step"] = ctx.print_expr(step)
        if attrs:
            kwargs["annotations"] = ctx.print_expr(attrs)
        return ast.Call(callee, args, kwargs)
    args = []
    if start != 0 or step != 1:
        args.append(ctx.print_expr(start))
    args.append(ctx.print_expr(end))
    if step != 1:
        args.append(ctx.print_expr(step))
    return ast.Call(ast.Id("range"), args)
```

**Assert** — assertion statement:

```python
def print_assert(node, trait, ctx, path):
    cond = ctx.print_expr(resolve(trait.cond, node), path.attr("cond"))
    if trait.message:
        msg = ctx.print_expr(resolve(trait.message, node), path.attr("message"))
        return ast.Assert(cond, msg)
    return ast.Assert(cond)
```

**With**, **While**, **If**, **Class** follow the same pattern: resolve
fields from the trait, enter scope, print prologue/body/epilogue.

### Binding Dispatch Rule

When printing an **Assign** whose RHS has a region-based trait (`If`,
`Func`), the printer wraps the assignment around the region:

```python
def print_assign(node, trait, ctx, path):
    rhs_node = resolve(trait.rhs, node)
    rhs_trait = get_trait(rhs_node)

    # Rule: If RHS is an If expression, hoist assignment into branches
    if isinstance(rhs_trait, If):
        return print_assign_if(node, rhs_node, rhs_trait, ctx, path)

    # Rule: If RHS is a Func, print as inline function definition
    if isinstance(rhs_trait, Func):
        return print_func(rhs_node, rhs_trait, ctx, path)

    # Rule: If RHS is void, print as expression statement
    if is_void(rhs_node):
        return ast.ExprStmt(ctx.print_expr(rhs_node, ...))

    # Default: lhs = rhs with optional type annotation
    lhs = print_value_def(resolve(trait.def_values, node), ...)
    rhs = ctx.print_expr(rhs_node, ...)
    return ast.Assign(lhs, rhs, annotation=...)
```

This handles Relax's VarBinding context-dependency (binding.cc:59-82)
without special traits.

### Annotation Intelligence

The printer may omit type annotations when they are inferable from
context.  This is a **printer-level** feature, not a trait parameter:

- Annotations on `call_tir`/`call_dps_packed` bindings: from `sinfo_args`
- Annotations on `Tuple` construction: from field types
- Annotations on `eGetItem`: from parent tuple type

The printer exposes a `should_annotate(var, rhs) -> bool` hook that
language modules can customize.

## Parser

### Value-Driven Dispatch

The parser evaluates expressions first, then dispatches on the result:

```
Parser sees:    with T.block("name") as [vi, vj]: ...
Step 1: Eval    T.block("name")  →  BlockFrame
Step 2: Call    BlockFrame.__ffi_text_parse__(parser_ctx, with_node)
```

Each language-module entry returns a **surface object** whose
`__ffi_text_parse__` handles the surrounding AST context.

### Surface Objects

Surface objects expose **position-specific** parse methods:

```python
class SurfaceObject:
    """Returned by language module entries (T.serial, R.dataflow, etc.)."""

    def __ffi_text_parse__(self, parser, node):
        """Generic dispatch.  Called by the parser core."""
        if isinstance(node, ast.For):
            return self.parse_for(parser, node)
        elif isinstance(node, ast.With):
            return self.parse_with(parser, node)
        elif isinstance(node, ast.Assign):
            return self.parse_assign(parser, node)
        elif isinstance(node, ast.FunctionDef):
            return self.parse_function(parser, node)
        elif isinstance(node, ast.ClassDef):
            return self.parse_class(parser, node)
        elif isinstance(node, ast.Expr):
            return self.parse_expr_stmt(parser, node)
        raise ParseError(f"Unexpected position for {type(self)}")
```

This replaces v0's single opaque `__ffi_text_parse__` method.  Benefits:
- `T.launch_thread` can implement `parse_assign`, `parse_expr_stmt`,
  and `parse_with` to support its multi-position syntax
- Unsupported positions fail with a clear error
- Auto-generation is straightforward per position

### Parser Core

```python
class IRParser:
    def __init__(self):
        self.var_table = VarTable()
        self.syntax_ctx = SyntaxContext()

    # --- Value-driven visitors ---

    def visit_For(self, node):
        return self._dispatch(self.eval_expr(node.iter), node)

    def visit_With(self, node):
        if len(node.items) == 1:
            return self._dispatch(
                self.eval_expr(node.items[0].context_expr), node)
        return self._multi_with(node)

    def visit_Assign(self, node):
        if _is_subscript_lhs(node):
            return self._subscript_store(node)
        return self._dispatch_or_bind(self.eval_expr(node.value), node)

    def visit_AnnAssign(self, node):
        return self._dispatch_or_bind(self.eval_expr(node.value), node)

    def visit_FunctionDef(self, node):
        for dec in node.decorator_list:
            if r := self._try_dispatch(self.eval_expr(dec), node):
                return r
        raise ParseError("Unrecognized decorator")

    def visit_ClassDef(self, node):
        for dec in node.decorator_list:
            if r := self._try_dispatch(self.eval_expr(dec), node):
                return r
        raise ParseError("Unrecognized decorator")

    def visit_Expr(self, node):
        self._try_dispatch(self.eval_expr(node.value), node)

    # --- Structural visitors (SyntaxContext) ---

    def visit_If(self, node):     return self.syntax_ctx.handle_if(self, node)
    def visit_While(self, node):  return self.syntax_ctx.handle_while(self, node)
    def visit_Return(self, node): return self.syntax_ctx.handle_return(self, node)
    def visit_Assert(self, node): return self.syntax_ctx.handle_assert(self, node)
    def visit_AugAssign(self, node): ...  # desugar to load + op + store
    def visit_Continue(self, node  ...
    def visit_Break(self, node):     ...
    def visit_Pass(self, node):      pass

    # --- Dispatch helpers ---

    def _dispatch(self, val, node):
        if hasattr(val, "__ffi_text_parse__"):
            return val.__ffi_text_parse__(self, node)
        raise ParseError(f"Expected IR construct, got {type(val)}")

    def _dispatch_or_bind(self, val, node):
        if hasattr(val, "__ffi_text_parse__"):
            return val.__ffi_text_parse__(self, node)
        self._default_bind(val, node)

    def _try_dispatch(self, val, node):
        if hasattr(val, "__ffi_text_parse__"):
            return val.__ffi_text_parse__(self, node)
        return None
```

### SyntaxContext

Structural constructs (`if`, `while`, `return`, `assert`) have
dialect-specific semantics.  `SyntaxContext` is set by the **language
module**, not derived from traits:

```python
@dataclass
class SyntaxContext:
    handle_if: Callable
    handle_while: Callable
    handle_return: Callable
    handle_assert: Callable
```

The function decorator's surface object pushes a SyntaxContext when
entering the function body:

```python
class PrimFuncDecorator(SurfaceObject):
    def parse_function(self, parser, node):
        with parser.set_syntax_ctx(TIR_SYNTAX):
            params = self._parse_params(parser, node)
            body = parser.visit_body(node.body)
            return PrimFunc(params=params, body=body, ...)
```

Each language module declares its SyntaxContext:

```python
TIR_SYNTAX = SyntaxContext(
    handle_if=tir_handle_if,         # static branching for bool, IR IfThenElse for PrimExpr
    handle_while=tir_handle_while,   # standard while loop
    handle_return=tir_handle_return,  # T.ret()
    handle_assert=tir_handle_assert,  # structured assert
)

RELAX_SYNTAX = SyntaxContext(
    handle_if=relax_handle_if,       # requires else, constructs relax.If, binds result
    handle_while=lambda p, n: raise_(ParseError("while not supported in Relax")),
    handle_return=relax_handle_return,  # R.func_ret_value()
    handle_assert=relax_handle_assert,
)
```

This replaces TVM's dialect-token dispatch stack.

### Parser Generation

The framework auto-generates surface objects from traits.  For each
language module entry, the trait metadata determines the parse logic:

```python
# ffi.lang.for_iter(TIRFor, kind="serial") generates:
class _SerialForIter(SurfaceObject):
    def __init__(self, *args, **kwargs):
        # args/kwargs carry the range call arguments: (start, end, step=...)
        self._start, self._end = parse_range_args(args, kwargs)
        self._step = kwargs.get("step", None)
        self._annotations = kwargs.get("annotations", None)

    def parse_for(self, parser, node):
        with parser.var_table.frame():
            loop_var = parser.create_var(node.target)
            parser.var_table.define(node.target.id, loop_var)
            body = parser.visit_body(node.body)
            return TIRFor(loop_var=loop_var,
                          min=self._start, extent=self._end - self._start,
                          step=self._step, kind=ForKind.SERIAL,
                          annotations=self._annotations, body=body)

# ffi.lang.with_ctx(SBlock) generates:
class _SBlockFrame(SurfaceObject):
    def parse_with(self, parser, node):
        with parser.var_table.frame():
            as_vars = self._create_iter_vars()
            if node.items[0].optional_vars:
                parser.bind_vars(node.items[0].optional_vars, as_vars)
            body = parser.visit_body(node.body)
            return SBlock(iter_vars=as_vars, body=body, ...)

# ffi.lang.decorator(PrimFunc) generates:
class _PrimFuncDecorator(SurfaceObject):
    def parse_function(self, parser, node):
        with parser.var_table.frame():
            with parser.set_syntax_ctx(TIR_SYNTAX):
                params = self._parse_params(parser, node)
                body = parser.visit_body(node.body)
                return PrimFunc(params=params, body=body, ...)
```

## Canonical Syntax

Each trait maps to a well-known Python syntax pattern.  This is the
**stable, roundtrip-safe form** that the auto-generated printer produces:

```python
# Value reference
${name}

# BinOp / UnaryOp
${lhs} + ${rhs}
-${operand}

# Load / Store
${source}[${indices}]
${target}[${indices}] = ${value}

# Assign
${def_values} = ${rhs}
${def_values}: ${type_ann} = ${rhs}

# Assert
assert ${cond}, ${message}

# Return
return ${value}

# Func
@${text_printer_kind}
def ${symbol}(${def_values}):
    ${prologue}
    ${body}

# For (range-based)
for ${def_values} in range(${start}, ${end}, ${step}):
    ${body}
# For (with kind)
for ${def_values} in ${text_printer_kind}(${start}, ${end}, step=${step}):
    ${body}
# For (generic, via region.def_expr)
for ${def_values} in ${def_expr}:
    ${body}
# For (with carry — reduction loop)
for ${def_values} in ${kind}(${range}):
    ${def_carry} = ${lang}.def_carry(${carry_init})
    ${body}
    yield ${carry_values}

# With
with ${def_expr} as ${def_values}:
    ${prologue}
    ${body}
# With (with carry — scoped carryover)
with ${kind}(${args}) as ${def_values}:
    ${def_carry} = ${lang}.def_carry(${carry_init})
    ${body}
    yield ${carry_values}

# ElementWise
with ${text_printer_kind}(${inputs}, ${outputs}) as (${body_args}):
    ${body}

# If
if ${cond}:
    ${then_body}
else:
    ${else_body}

# While
while ${cond}:
    ${body}

# Module
@${text_printer_kind}
class ${symbol}:
    ${prologue}
    ${members}

# Class
@${text_printer_kind}
class ${symbol}(${bases}):
    ${prologue}
    ${members}
```

### Sugar vs Canonical

Traits generate **canonical** syntax.  Alternative spellings are surface
sugar, handled by Level 2 overrides or surface codecs:

| Sugar | Why not canonical |
|-------|-------------------|
| `T.grid(4, 4)` | Desugared to nested serial loops in final IR |
| `T.axis.remap("SSR", ...)` | Desugared to individual axis bindings |
| `T.macro` / `R.macro` | Macro body inlined; definition lost |
| `T.meta_var` | Parse-time-only; not in final IR |
| `R.emit_te` | Desugared to `call_tir` |
| `T.comm_reducer(lambda ...)` | Lambda lowered away |

Sugar is supported via Level 2 overrides and is never required for the
canonical printer to work.

## Module Parsing

Module-level classes (`@I.ir_module`) require multi-pass parsing.  This
is handled by the module decorator's surface object, not by the Class
trait alone:

```python
class IRModuleDecorator(Surject):
    def parse_class(self, parser, node):
        # Pass 1: module attrs, global infos, extern members
        for stmt in node.body:
            if is_non_function(stmt):
                self._parse_preamble(parser, stmt)
        # Pass 2: forward-declare all functions
        for stmt in node.body:
            if isinstance(stmt, ast.FunctionDef):
                gvar = self._declare_function(parser, stmt)
                parser.var_table.define(stmt.name, gvar)
        # Pass 3: parse function bodies
        for stmt in node.body:
            if isinstance(stmt, ast.FunctionDef):
                self._parse_function(parser, stmt)
        return IRModule(...)
```

This is Level 2 code on the surface object, not on the IR class.

Below is a complete example applying traits to a graph-level IR.

operations — a different flavor from TIR's loop-heavy programs.

### IR Definitions with Traits

```python

    dtype: str

| `Tuple` | L0 | — | Tuple literal |
| `TupleGetItem` | L0 | — | Subscript notation |
| `ShapeExpr` | L0 | — | R.shape([...]) call |
| `Constant` | L0 | — | R.const(...) |
| `PrimValue` | L0 | — | R.prim_value(...) |
| `StringImm` | L0 | — | R.str(...) |
| `DataTypeImm` | L0 | — | R.dtype(...) |
| `BindingBlock` | L0 | — | Print bindings in order |
| `ExternFunc` | L0 | — | R.ExternFunc(...) |
| `SeqExpr` | — | — | Compound body flattening; framework handles |
| `Call` | L2 | — | `ctx.call` + variant dispatch |
| StructInfo (6 types) | L2 | — | `ctx.call` + `ctx.elide` per type |

**Summary**: 6 nodes at Level 1, 9 at Level 0, 2 at Level 2, 1
framework-handled.  Level 1 covers the main structural nodes.  Only
Relax Call variants and StructInfo types need Level 2.

### MLIR Operations (if defined via @py_class)

| Pattern | Level | Trait | Notes |
|---------|-------|-------|-------|
| `arith.*` (add, sub, ...) | L1 | BinOp/UnaryOp | Operator syntax |
| `memref.load/store` | L1 | Load/Store | Subscript syntax |
| `func.func` | L1 | Func | Direct mapping |
| `scf.for` (simple) | L1 | For | `start`/`end`/`step` |
| `scf.for` (with iter_args) | L1 | For + `def_carry` | Carry fields handle loop-carry |
| `scf.if` | L1 | If | `def_carry`/`carry_init` for typed results |
| `scf.while` | L2 | — | Two named regions (before/after) |

| Most other ops | L0 | — | Default `TypeKey(fields...)` |

**Note**: MLIR operations typically use MLIR's own ODS and
`assemblyFormat` for MLIR-native text.  The traits system would apply
only if someone wanted Python-style tex its two-region structure (condition region
+ body region) has no single-skeleton trait.  All other SCF ops are
now covered: `scf.for` via `For` + `def_carry`, `scf.if` via `If` +
carry, `linalg.generic` via `ElementWise`.

### cuteDSL

cuteDSL uses MLIR's native text format and does not define `@py_class`
IR nodes.  A comprehensive validation against all ~540 MLIR operations
across 10 cuteDSL dialects (Tensor IR, Collective, LIR, Cutlass, Cute,
CuteNvGPU, Core, Iket, Feynman, DebugInfo) confirms:

| Structural Kind | Count | Level 0 | Level 1 | Level 2 |
|-----------------|-------|---------|---------|---------|
| FUNCTION | 3 | 0 | 3 (Func) | 0 |
| REGION_CF | 17 | 0 | 15 (With ~10, For/With+carry ~5) | 2 |
| REGION_BODY | 10 | 0 | 7 (ElementWise ~5, With ~2) | 1 |
| TERMINATOR | 12 | 0 | 12 (Return) | 0 |
| PURE_VALUE | ~380 | ~365 | ~15 (BinOp/UnaryOp) | 0 |
| SIDE_EFFECT | 147 | 141 | 6 (Load/Store) | 0 |
| ALLOC | 13 | 13 | 0 | 0 |
| CALL | 3 | 3 | 0 | 0 |
| REGION_GRAPH | 1 | 0 | 0 | 1 |
| **Total** | **~540** | **~487 (90%)** | **~58 (11%)** | **~4 (<1%)** |

**Level 0 handles 90% of ops** — hardware intrinsics, layout algebra,
factory ops, and synchronization primitives render naturally as
`dialect.op(field=value, ...)`.

**Level 1 now covers all major MLIR structural patterns**:

- **SingleBlock+NoTerminator scope ops → `With`** (~10 ops):
  `cutlass.pipeline.produce`, `.consume`, `.producer_tail`,
  `cutlass.async.exec`, `lir.with_structured_alloc`, `.with_union_alloc`,
  `lir.detail.initialize_params`, `.configure_launch`, `.can_implement`,
  `.allocate_device_workspace`.  These are literally `with op(): body`.

- **Loop-carry ops → `For`/`With` + `def_carry`/`carry_init`** (~5 ops):
  `collective.for_each_tile` (For + carry), `collective.mma`,
  `.epilogue`, `.tile_reduce`, `.pipeline` (With + carry).  The body
  yields values carried to the next iteration or returned as results.

- **Body-region computation → `ElementWise`** (~5 ops):
  `lir.generic`, `.warp_reduce`, `.atomic_generic`,
  `collective.reduce`, `.ref.generic`.  Inputs/outputs + body describes
  per-element computation.

- **Queue operations → `With`** (~2 ops):
  `collective.queue.put`, `.queue.get`.  Body region scoped by queue op.

**Remaining L2** (~4 ops, genuinely multi-region or graph-structured):

| Op | Why L2 |
|----|--------|
| `scf.while` | Two named regions: condition (before) + body (after) |
| `nv_tensor_ir.sdpa` | Two optional body regions: score_mod + mask_mod |
| `collective.task_graph` | Graph region (unordered, no terminator) |
| `collective.agent` (if multi-region) | Parent constraint + mixed region semantics |

These are rare, varied patterns where a dedicated trait would not
generalize.  They use L2 with `ctx` helpers.

**MLIR-specific caveats** (not affecting `@py_class` nodes):
- Implicit regions: MLIR regions are not dataclass fields.  When
  porting to `@py_class`, regions become explicit fields and traits
  apply naturally.
- `FunctionType`-encoded params: MLIR's Func uses FunctionType to
  encode parameters; `@py_class` would have an explicit `params` field.

## Comparison with Existing Systems

### vs TVM TVMScript (~5000 LOC per dialect)

TVMScript requires:
- Hand-written C++ printer per node type (IRDocsifier dispatch)
- Hand-written Python parser per node type (dispatch.register handlers)
- Global dispatch table + isinstance chains (bind_assign_value)
- Dialect token stack management

This design replaces:
- **Dispatch table** → value-driven dispatch via `__ffi_text_parse__`
- **isinstance cha type's surface object handles its own parsing
- **Token stack** → SyntaxContext set by function decorator
- **C++ printer boilerplate** → auto-generated from traits

**Estimated reduction**: Level 0+1 nodes need zero printer/parser code.
Level 2 nodes need ~50% less code (framework handles scope, variable
management, frame stacking).  For a TIR-scale dialect (~32 nodes):
~16 Level 1 + 10 Level 0 + 5 Level 2 = ~95% code reduction.

### vs MLIR ODS

| Aspect | MLIR ODS | This Design |
|--------|----------|-------------|
| Definition language | TableGen | Python decorators |
| Target format | MLIR assembly | Python-style text |
| Auto-generation | `assemblyFormat` DSL | Trait + `text_*` params |
| Fallback | `hasCustomAssemblyFormat` | Level 2 override |
| Fallback rate | ~30% of operations | ~16% of TIR nodes |
| Trait system | C++ traits (SingleBlock, etc.) | Python traits (For, If, etc.) |
| Region model | Explicit block arguments + terminators | Python scoping + def_values |
| Loop-carry | Block arguments + yield | `def_carry` / `carry_init` + `yield` |

MLIR ODS is more expressive for MLIR assembly (optional groups, oilist,
custom directives).  This design is more natural for Python-style text
(Python syntax IS the format).  They serve different format families
and complement each other.

## Summary

| Layer | Concern | Mechanism |
|-------|---------|-----------|
| Semantic | What the IR node IS | `@ffi.traits.For(region=...)` |
| Semantic | Field references | `"$field:"` / `"$method:"` / `"$global:"` |
| Semantic | Loop-carry reduction | `For.def_carry` / `carry_init`, `With.def_carry` / `carry_init` |
| Semantic | Body-region computation | `ElementWise(inputs, outputs, region)` |
| Text L0 | Default printer/parser | Auto from `@py_class` fields |
| Text L1 | Syntax mapping | Trait decorator + default rendering |
| Text L1 | Custom callee | `text_printer_kind="$lang.prim_func"` |
| Text L1 | Region prologue/epilogue | `text_printer_pre` / `text_printer_post` |
| Text L2 | Irregular syntax | `__ffi_text_print__` / `__ffi_text_parse__` |
| Text L2 | Convenience methods | `ctx.call`, `ctx.elide`, `ctx.lambda_`, `ctx.scope`, `ctx.assign` |
| Framework | Transparent wrappers | Printer delegates to inner node (e.g. SBlockRealize) |
| Framework | Compound body flattening | Func trait flattens SeqExpr bodies inline |
| Parser | Dispatch model | Value-driven: `result.__ffi_text_parse__(parser, node)` |
| Parser | Structural rules | `SyntaxContext` on language module |
| Parser | Surface objects | Position-specific: `parse_for`, `parse_with`, etc. |
| Parser | Auto-generation | From trait metadata per surface object type |
| Parser | Module parsing | Multi-pass via module decorator surface object |
| Text | Language module | `@ffi.lang` + `tvm-ffi-stubgen` |
| Text | Binding context | Printer rule: Assign wrapping for If/Func RHS |
| Text | Annotation omission | Printer hook: `should_annotate(var, rhs)` |
